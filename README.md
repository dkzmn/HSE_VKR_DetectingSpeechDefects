```
# Полезное

# Все переменные из .env -> в bash export
export $(grep -v '^#' .env | xargs) 
```
## 1. Создание и анализ датасета

В блокноте [experiments/01_create_dataset.ipynb](experiments/01_create_dataset.ipynb)
- Скачиваем архив с аудиозаписями с Яндекс облака (датасет предоставленный дипломным руководителем)
- Удаляем пустые файлы и дубли
- Выделяем контрольные буквы из названий файлов
- Переименовываем файлы (чтобы не было кириллицы в названиях)
- Сохраняем датасет в [data/dataset.csv](data/dataset.csv)

## 2. Получение текстов скороговорок из файлов

В блокноте [experiments/02_get_texts.ipynb](experiments/02_get_texts.ipynb)
- Транскрибируем все аудиофайлы с помощью Whisper
- Исправляем ошибки с помощью Perplexity API
- Ищем одинаковые скороговорки с помощью SequenceMatcher
- Вручную исправляем те записи, которые не удалось исправить на предыдущих шагах (их было около 100 из-за плохой дикции на записях, галюцинаций Whisper)
- Сохраняем тексты скороговорок с контрольными буквами в [data/tongue_twisters.csv](data/tongue_twisters.csv)

## 3. Определение количества дикторов на записях

В блокноте [experiments/03_get_speakers.ipynb](experiments/03_get_speakers.ipynb)
 - Определяем количество дикторов на каждой аудиозаписи и добавляем в качестве нового признака

## 4. Анализ получившегося датасета

В блокноте [experiments/04_analyse_dataset.ipynb](experiments/04_analyse_dataset.ipynb)
 - Анализ всего датасета
 - Подготовка картинок с основными параметрами

 ## 5. Заливка данных в Yandex S3 
  - На данном этапе датасет полностью готов для экспериментов
  - Данные загружены в S3

## 6. Проведение экспериментов

```
dvc pull
cd experiments
pip install -r requirements.txt
```
 - Если `dvc pull` ругается на права можно так:
```
dvc pull -r http_yandex
```

 - После этого можно запускать блокноты из вложенных папок (1 папка - 1 эксперимент)
 - На первоначальном этапе использовался mlflow, но от него пришлось отказаться, так как эксперименты проводились одновременно на 3 разных машинах и синхронизация БД mlflow создавала проблемы. 
 - Каждый эксперимент создает result.csv и test_predictions.csv в своей папке, которые потом анализируются в отдельных блокнотах.
 - DL эксперименты также создают у себя в папках: best_ckpt.pt (лучший чекпоинт), threshold.json (оптимальный порог классификации, если используется) training_curves.png (графики обучения: лосс и f1_bad_val по эпохам)

## 7. Анализ результатов

В блокноте [experiments/05_results_summary.ipynb](experiments/05_results_summary.ipynb)
 - Сбор и анализ результатов всех экспериментов из result.csv
 - Выбор лучшей модели по метрике F1_bad

 ## 8. Анализ покрытия тестовой выборки

В блокноте [experiments/06_diversity_analysis.ipynb](experiments/06_diversity_analysis.ipynb)
 - Сбор и анализ результатов всех из test_predictions.csv 
 - Поиск тройки моделей которые максимально покрывают тестовую выборку, для проведения дополнительного эксперимента с объединением эмбеддингов.
